Plot Emu Trackdata

Description

A general purpose routine for plotting trackdata on a single plot. Tracks can be aligned at an arbitrary position, length normalised or averaged.

Usage

function(dataset, labs = NULL, offset = 0, ref.time = NULL,
         average = F, main = "", xlab = "time (ms)", ylab = "",
         xlim = NULL, ylim = NULL, cex = 0.5, linetype = F,
         normalise = F, colour = T, legn = "tl", axes = T, 
         amp = F, n = 20)

Arguments

dataset An Emu trackdata object.
labs A label vector with one element for each row in dataset.
offset A number between 0 and 1, segments are aligned on the plot with this proportion of their duration corresponing to zero.
ref.time A vector of one time per segment, an alternative to offset, segments will be lined up on the plot with these times at zero.
average If T, the data for each unique label in labs is averaged.
normalise If T, the data for each segment will be time normalised so that all segments appear to have the same length on the plot.
main A main title for the plot.
xlab A label for the x-axis.
ylab A label for the y-axis.
xlim A vector of the minimum and maximum limits of the x-axis.
ylim A vector of the minimum and maximum limits of the y-axis.
cex The cex parameter passed to par to control text size on the plot.
linetype
colour colour and linetype can be either T, F or a vector. If they are F then the plot will use a single colour or linetype. If they are T then the plot will use a different colour/linetype for each line. If they is a vector then it should consist of one label for each element to be plotted, the colour/linetype will be selected according to this label.
legn The position to draw the legend, one of "tl" (top-left), "tr" (top-right), "bl" (bottom-left) or "br" (bottom-right).
axes If T, axes will be drawn on the plot.
amp ~~Describe amp here~~

Details

This function is a general purpose plotting function for track data generated by the track function. Data for each segment is lined up at some point along its length (eg. at the start, midpoint or end) and possibly normalised or averaged before being plotted. Normalisation consists of stretching each segment out to have the same number of samples as the longest segment; the stretching is done with the Emu function linear. Averaging takes account of the different lengths of the segments in computing the average for each time point.

Value

None.

Examples


## assumes a database called demo is available on your system and that
## the Emu system is installed. 

# find all Phonetic vowels in the database
segs <- emu.query("demo", "*", "Phonetic=A|E|I|O|U|V")

# get formant data for entire tracks, returns trackdata
data.all <- emu.track( segs, "fm" )
# plot the data aligned at the midpoint
dplot( data.all, label(segs), offset=0.5, normalise=T, main="Formant data for vowels")
